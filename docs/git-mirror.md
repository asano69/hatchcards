# Design

- ラップトップで編集したマークダウンファイルは、Gitにプッシュすることになる。
- サーバは、あらかじめ設定したリポジトリから定期的にミラーする。
- go-gitでのミラー処理をPocketBaseのバイナリに直接組み込み、PocketBase内蔵のcronで動かす
- 強制的にミラーをトリガーするためのAPIをつくる。


## 秘密管理

- DBに保存する秘密情報を暗号化 → 使用時にメモリ上でのみ復号、という要件であれば、標準ライブラリ（AES-GCM）で自前実装するのが依存関係も少なく、コードも短く、メンテナンスしやすいはずです。マスターキーの管理さえ別途（KMSや環境変数、OSキーリング）きちんとしていれば、これで十分実用的です。crypto/aes + crypto/cipher(GCM) だけで十分に実用的です。ロジックが数十行で完結し、依存の脆弱性リスクやAPI変更の心配もありません。
- DBにはEncryptの戻り値をそのまま（byteaやbase64で）保存。使う直前にDecryptでメモリに展開し、gitのSSH鍵やトークンとして利用。使い終わったらfor i := range plaintext { plaintext[i] = 0 }でゼロクリアするとより安全です。マスターキー（32バイト）自体は環境変数から取得する。
