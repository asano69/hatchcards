# Effective Hatchards

特徴：
- hashcadsのよいところは、複数のノートに分散したフラッシュカードを、スクリプトで集約してデッキを作成できること。
- カードが直感的で見通しがよいこと。アナログノートに単語帳を作成するような感覚でカードを作成することができる。
- Ankiの場合、カード自体がDBに保存され、カードの状態と密結合になっているためにスクリプトなどで自動化しずらい。

設計：
- hatchardsは、暗記するべき内容と、フラッシュカードの状態を分離し、状態のみをDBで管理する設計になっている。
- フラッシュカードはマークダウンで管理するために、ターミナルから編集できる。これによってワークフローがよりシンプルになる。

考察：
-   hashcardの特徴上、カードはどこにでも保存できる。学習用のマークダウンノートの中に決められた記法でカードを作成すれば、それをpythonスクリプトのようなものでパースすることによって、カードを作成するやり方がよいだろう。
-   学習用のノートと同じマークダウンファイルに暗記カードを作成する。
-   ものを暗記するには理解が重要で、背景知識と同じファイルに、フラッシュカードを保存するのは合理的。フラッシュカード自体はどこにでも埋め込むことができ、それをスクリプトで集約する設計が、ハッシュカードと相性がよい。
